Spec Detail Page Layout

How should the expanded spec detail page organize stage outputs, edges, and conversations?

A

Single Scrollable Page

All sections stacked vertically: metadata → stage outputs → edges → conversations. Simple, matches CLI output order. Good for reading top-to-bottom. Could get long for fully-decomposed specs.

Pros

  • Simple to implement
  • Matches CLI show output
  • Works on mobile
  • Familiar pattern

Cons

  • Long scroll for complete specs
  • Hard to jump between sections
B

Tabbed Sections

Metadata always visible at top. Tabs for: Overview | Spark | Shape | Specify | Decompose | Edges | Conversations. Only shows tabs for stages the spec has completed.

Pros

  • Compact — no scrolling
  • Quick navigation between sections
  • Only shows relevant stages

Cons

  • More complex component
  • Can't see everything at once
  • Tab state management
C

Collapsible Accordion

All sections visible as headers. Click to expand/collapse. Metadata always expanded. Stage sections auto-expand for current stage, collapsed for past stages. Edges and conversations collapsed by default.

Pros

  • See all section headers at a glance
  • Expand only what you need
  • Current stage auto-highlighted
  • Medium complexity

Cons

  • Slightly more complex than A
  • Requires expand/collapse state
