Graph Visualization Library

Which rendering approach fits best for an interactive spec dependency graph?

A

D3-force + SVG

Force-directed layout — nodes push/pull each other into natural clusters. The classic interactive graph look.

login-api
token
gateway
crypto
Nodes float organically • drag to rearrange • zoom/pan

Pros

  • Most intuitive for exploring relationships
  • Handles organic, non-hierarchical graphs well
  • D3 is battle-tested, huge ecosystem

Cons

  • Can be chaotic with 50+ nodes
  • Layout isn't deterministic (different each load)
  • Heavier JS bundle (~50KB)
B

ELK / Dagre Layered Layout

Hierarchical/layered layout — nodes arranged in layers by dependency depth. Like a refined Graphviz output but interactive.

gateway
↓
login-api
webhooks
↓
token-storage
crypto-utils
Layers = dependency depth • deterministic layout • click to expand

Pros

  • Dependency flow is immediately obvious (top → bottom)
  • Deterministic — same graph = same layout every time
  • Scales better to large graphs (clear structure)
  • Natural fit for spec dependency graphs

Cons

  • Less "organic" feel than force-directed
  • ELK is larger (~200KB) but Dagre is lighter (~30KB)
  • Cross-edges (relates_to) can create visual clutter
C

Mermaid.js (rendered)

Generate Mermaid diagrams — server sends Mermaid syntax, browser renders it. Simplest to build, familiar output.

graph TD
  gateway --> login-api
  gateway --> webhooks
  login-api --> token-storage
  login-api --> crypto-utils
  webhooks --> token-storage
Text-based • click nodes for details • limited interactivity

Pros

  • Fastest to implement (generate text, render client-side)
  • CLI can also emit Mermaid for README/docs
  • Lightweight, familiar format

Cons

  • Limited interactivity (no drag, limited click handling)
  • Doesn't scale well past ~30 nodes
  • Less "wow factor" for demos
  • Hard to customize node styling

My recommendation: B (ELK/Dagre layered) — spec dependency graphs have natural hierarchy (dependency depth). A layered layout makes the flow immediately obvious, is deterministic (great for screenshots/embedding), and scales better. We can add Mermaid CLI export as a bonus without it being the primary visualization.
