<objective>
Deliver the AUTH-04 OAuth 2.1 resource-server protocol surface: the RFC 9728 Protected Resource
Metadata endpoint and the WWW-Authenticate challenge on unauthenticated MCP requests, so standard
MCP clients auto-discover the authorization server(s) (MCP Authorization spec rev 2025-06-18,
D-05.1/D-05.2/D-07).

Scope (Pattern 3):
1. `GET /.well-known/oauth-protected-resource` → 200 application/json with `resource` = the
   canonical MCP resource URI and `authorization_servers` = every configured interactive/OAuth IdP
   issuer (advertise all, D-07). Public, unauthenticated.
2. A `/mcp/`-only auth wrapper that emits `WWW-Authenticate: Bearer resource_metadata="…"` on a
   401, replacing the bare 401 that `auth.RequireAuth` writes — scoped so `/api/*` and ConnectRPC
   401s are untouched (D-08, Pitfall 4).
3. Canonical resource URI: derive from `cfg.Auth.OIDC.MCPResourceURI`, defaulting to
   `<base_url>/mcp`, and validate https + no-fragment at startup (startup-fatal, RESEARCH OQ1/A2).

Purpose: The discovery handshake an MCP client needs before it can obtain and present a
resource-bound token; the token-validation half lands in Plan 04.
Output: The metadata handler + challenge wrapper mounted in serve.go, green under `task test`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → /mcp/ and /.well-known/oauth-protected-resource | External agent discovers the AS and presents (or lacks) a token |
| operator config → canonical resource URI | Misconfiguration would make clients reject tokens or advertise a wrong AS |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-03-01 | Spoofing | canonical resource URI | high | mitigate | resource in metadata MUST equal the identifier clients fetch it by (RFC 9728 §3.3); validated https+no-fragment at startup (fatal); asserted by the metadata test. |
| T-03-03-02 | Denial of Service (lockout) | challenge scope (D-08) | high | mitigate | WWW-Authenticate is emitted ONLY by the /mcp/ wrapper; RequireAuth/ConnectRPC/`/api/*` 401s unchanged — static-credential clients still get a plain 401 and keep working (Pitfall 4). |
| T-03-03-03 | Information Disclosure | metadata endpoint | low | accept | The endpoint is intentionally public (unauthenticated discovery, like /api/auth/oidc/providers); it exposes only issuer URLs already known to clients. |
| T-03-03-04 | Elevation of Privilege | token passthrough | high | mitigate | Confirm the loopback mcpClient hop is internal same-audience to SpecGraph's own ConnectRPC (A3), not an upstream API — no forbidden passthrough. |
| T-03-03-SC | Tampering | package installs | high | accept | No new packages this phase (RESEARCH Package Legitimacy Audit: none). |
