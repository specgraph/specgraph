<objective>
Create a new `internal/authoring/load` package that parses friendly snake_case YAML into the four authoring funnel stage proto outputs, mirroring the existing `internal/constitution/load` pipeline. This is the reusable transform layer the MCP funnel handlers (plan 04) will call so an MCP-only agent authors from friendly YAML instead of raw protojson (D-01, D-02, D-03). Enum mappers enforce the #1002 correctness fix: unknown enum values error, never silent-write (D-04).

Purpose: Give the funnel `author.<stage>` handlers a friendly-input parser with the same "server is the schema" property the constitution path already has, without any proto change.
Output: `internal/authoring/load/load.go` (parsers + enum mappers) and `load_test.go` (mapping + rejection coverage — Wave 0 test #3 from 06-VALIDATION.md).
</objective>
