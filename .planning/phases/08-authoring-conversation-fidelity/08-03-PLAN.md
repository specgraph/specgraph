<objective>
Replace the CLI synthetic-placeholder escape hatch with a real conversation input (D-04/D-05). Delete `cliSyntheticExchanges` and give the stage commands a shared `--conversation <file.json>` loader (`--conversation -` for stdin) that accepts a bare JSON array of `ConversationExchange` objects — the same shape the MCP `author` tool accepts — so there is one validation contract. shape/specify/decompose/approve require it; spark keeps it optional (D-01).

Purpose: Close the "CLI stamps a fake conversation" hole so real exchanges are required everywhere (the user-accepted fidelity-over-convenience tradeoff). The server (Plan 01/existing) is the enforcement authority; the CLI simply supplies real input.
Output: A shared `loadConversationFlag` helper, five rewired stage commands, deleted placeholder helper, and CLI unit tests.
</objective>
