<objective>
Collapse the MCP recording surface to a single inline-with-save path (CONV-01 success criteria #1/#2; the #906 root cause is the skippable standalone record action). Changes to the MCP surface: (1) `handleApprove` threads required `exchanges` into `ApproveRequest` (D-02, mirroring `handleShape`); (2) `handleSpark` forwards OPTIONAL `exchanges` into `SparkRequest` when the agent provides them (D-01 parity with the server, which records spark exchanges when present — currently the MCP path drops-when-provided; review R2 #3); (3) remove the `conversationTool` `record` action so `list` is the only action (D-06, Phase 7 single-source-of-truth precedent), keeping retrieval intact (criterion #4); (4) flip the `author` tool Description and `specgraph-authoring` SKILL.md so approve is taught as requiring exchanges (D-09, Pitfall #5).

Purpose: After this plan, an MCP-only agent has no "forgot to record" path — inline-with-save is the only way, approve joins the enforced stages, and agent-supplied spark exchanges are no longer silently dropped.
Output: Updated MCP tool handlers + descriptions + skill; extended approve mock so unit tests can verify threading; pruned record-action unit tests.
</objective>
