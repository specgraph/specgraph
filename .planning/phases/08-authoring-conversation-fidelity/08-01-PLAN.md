## Trust Boundaries

| Handler -> PostgreSQL (RunInTransaction) | Domain conversation entry crosses into durable storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-01 | Tampering / DoS | `conversation_exchanges` volume/size on Approve | medium | mitigate | Reuse `authoring.ValidateExchanges` caps (MaxConversationExchanges=100, MaxExchangeContentLen=4096) verbatim in the accept branch; do not relax (V5). |
| T-08-02 | Information Disclosure | Approve handler error surface | low | mitigate | Route tx errors through existing `h.stageError` sanitization; tests assert `connect.Code*` not messages (Pitfall #4). |
| T-08-03 | Elevation / Access Control | New accept record path | medium | mitigate | Accept branch inherits the existing `scopeStore(ctx, h.scoper)` project scoping already applied in Approve — do not bypass it (V4). |
| T-08-04 | Tampering | Partial-write (stage approved but no conversation) | high | mitigate | RecordConversation op runs inside the existing accept `runInTxOrSequential` (ADR-004) so approval + conversation commit or roll back together — a missing conversation cannot silently pass. |

No package installs in this plan (no npm/pip/cargo) — supply-chain threat T-08-SC not applicable. T-08-04 is the only high-severity item and is mitigated by the atomic-transaction requirement enforced in Task 2.
