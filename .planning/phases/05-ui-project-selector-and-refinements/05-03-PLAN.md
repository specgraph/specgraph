<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → client | `specgraph-project` and mode-watcher key seed selection/theme |
| client Select → header | Project choice becomes `X-Specgraph-Project` on every RPC |
| client → server RPC | ConnectRPC POSTs cross into the Go server |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-04 | Elevation of Privilege | `X-Specgraph-Project` header chosen client-side | high | accept | Header is advisory; server authorizes via authenticated identity + `storage.Scoper` (ProjectMiddleware), not the raw client header. Switching only re-scopes to projects the identity may already access. |
| T-05-05 | Information disclosure | stale previous-project data lingering after switch | medium | mitigate | `invalidateAll()` re-runs every load; Wave 3 pages return to skeleton then new-project data. Shell exposes no page data itself. |
| T-05-06 | Tampering | FOUC inline script reads localStorage | low | accept | Reads only a theme string; toggles a CSS class; no injection surface (value not interpolated into HTML). |
</threat_model>
