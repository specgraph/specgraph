# Phase 5 — UI Design Contract

> Visual and interaction contract for the SpecGraph web UI (`web/`, SvelteKit + Svelte 5 runes). Covers the full shadcn-svelte migration (D-12/D-13/D-14) **and** the project-selector / switching refinements (D-01–D-11). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope reminder:** This contract governs the *visual and interaction* layer. The data-loading mechanism (moving fetches into `load()` + `invalidateAll()` on switch, D-01/D-02) is the planner's job — this spec defines the loading / empty / error *visual states* those loads must render.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn-svelte (not yet initialized — installed this phase per D-12) |
| Preset | Default aesthetic, base color **Slate** (planner's `components.json`: `"baseColor": "slate"`, `"style": "default"`, `"tailwind.css": "src/app.css"`, `"aliases.components": "$lib/components"`, `"aliases.ui": "$lib/components/ui"`) |
| Component library | `bits-ui` (Svelte 5 primitive layer under shadcn-svelte) |
| Icon library | `lucide-svelte` |
| Utilities | `tailwind-variants` + `tailwind-merge` + `clsx` (via generated `cn()` in `$lib/utils.ts`) |
| Theming | shadcn CSS-variable tokens in `src/app.css`; light + dark via `mode-watcher` (`<ModeWatcher />` in `+layout.svelte`), `darkMode: "class"` |
| Font | Tailwind default sans stack mapped to `font-sans` (`ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, sans-serif`); `font-mono` for slugs / code / diffs |
| Build constraint | Must stay a static bundle for `@sveltejs/adapter-static` → `web/embed.go`. Tailwind runs at Vite build; theming is client-side (CSS vars + `mode-watcher` no-FOUC script). Keep Svelte 5 runes. Confirm `export const ssr = false` / prerender fallback stays compatible. |

**shadcn-svelte components to add (registry: shadcn-svelte official):**
`button`, `card`, `dialog`, `alert-dialog`, `dropdown-menu`, `select`, `tabs`, `table`, `accordion`, `badge`, `input`, `separator`, `skeleton`, `tooltip`, `sonner` (toast, optional for switch feedback).

**Base color rationale:** The current plain-CSS palette already uses Tailwind slate values (`#64748b`, `#475569`, `#e2e8f0`, `#f8fafc`). Slate is the seamless continuation of that neutral and reads well in both light and dark mode. The navy `#1a1a2e` nav and `#2563eb` blue accent (D-13) are retired in favor of Slate tokens.

## Spacing Scale

Declared values (multiples of 4 — maps directly to Tailwind spacing tokens):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `1` | Icon-to-label gaps, badge inner padding |
| sm | 8px | `2` | Compact element spacing, table cell padding-y |
| md | 16px | `4` | Default element spacing, card padding, gap between controls |
| lg | 24px | `6` | Section padding, card internal gaps |
| xl | 32px | `8` | Layout gaps between dashboard rows |
| 2xl | 48px | `12` | Major section breaks |
| 3xl | 64px | `16` | Page-level top/bottom spacing |

**Layout:** main content `max-w-[1400px] mx-auto p-6`. Nav bar `px-6 py-3`. Dashboard row grid gap `gap-6`.

Exceptions: interactive nav/select/button hit targets must be ≥ 36px tall (shadcn `size="sm"` = h-9 = 36px; default = h-10 = 40px). Icon-only toggle (theme switch) uses `size="icon"` (h-9 w-9 = 36px).

## Typography

Exactly 4 roles, exactly 2 weights (400 regular, 600 semibold).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 | 1.5 | Table rows, card body, list items, form values |
| Label | 12px (`text-xs`) | 600 | 1.4 | Table headers, badges, muted meta labels, breadcrumb |
| Heading | 20px (`text-xl`) | 600 | 1.2 | Page `h1` (Dashboard, Constitution, Spec/Decision titles) |
| Display | 30px (`text-3xl`) | 600 | 1.2 | Large stat numbers in `StatsBar` |

Section sub-headings (`h3` inside accordions) use Body size at weight 600 — no additional role. Muted secondary text uses `text-muted-foreground`.

## Color

shadcn Slate tokens (CSS variables from `app.css`). The 60/30/10 split maps to the token roles below; both light and dark are defined by the theme.

| Role | Token | Light | Dark | Usage |
|------|-------|-------|------|-------|
| Dominant (60%) | `--background` / `--foreground` | white / slate-950 | slate-950 / slate-50 | Page background, primary text |
| Secondary (30%) | `--card`, `--muted`, `--secondary`, `--border` | slate-50 / slate-100 / slate-200 | slate-900 / slate-800 | Cards, nav bar, table header rows, dividers, hover states |
| Accent (10%) | `--primary` | slate-900 | slate-50 | See reserved-for list below |
| Destructive | `--destructive` | red-500/600 | red-900/500 | Destructive confirmations only (revoke key / sign out) |

**Accent (`--primary`) reserved for — explicit list, never "all interactive elements":**
1. Primary buttons (e.g. modal confirm, retry-on-error button)
2. Active nav link (current route)
3. Focus rings (`--ring`) on inputs, selects, buttons
4. The active-project indicator emphasis (D-11)
5. Hyperlinks in tables/breadcrumbs (spec/decision slugs)

Everything else (secondary buttons, badges, cards, borders) uses secondary/muted tokens, not accent.

### Categorical badge palette (carved-out exception)

The Constitution layer badges are *categorical data encoding*, not part of the 60/30/10 UI accent budget. Preserve four fixed, colorblind-distinguishable hues via a `Badge` variant map (D-10 requires these to survive project switches). Use fixed Tailwind color utilities (light / dark pairs), NOT the theme accent:

| Layer | Light bg / text | Dark bg / text |
|-------|-----------------|----------------|
| User | `bg-blue-100 text-blue-800` | `bg-blue-950 text-blue-300` |
| Org | `bg-amber-100 text-amber-800` | `bg-amber-950 text-amber-300` |
| Project | `bg-green-100 text-green-800` | `bg-green-950 text-green-300` |
| Domain | `bg-violet-100 text-violet-800` | `bg-violet-950 text-violet-300` |

**Merged / Layer view badge (D-10):** the "Merged" vs single-"Layer" badge uses the neutral shadcn `Badge variant="secondary"`. Both the Merged/Layer badge and the four layer badges MUST re-render correctly after a project switch (they are derived from the reloaded `provenance` array).

Antipattern cards keep a `border-l-destructive` accent; principle cards keep `border-l-primary`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary control | Project selector — `aria-label="Select project"`; placeholder never shown (a project is always resolved per D-04) |
| Theme toggle | Icon button, `aria-label="Toggle theme"` (sun/moon lucide icons) |
| Empty state heading (zero projects, D-07) | **No projects found** |
| Empty state body (zero projects) | Create a project with the SpecGraph CLI or authoring flow, then reload this page. |
| Empty state — Constitution (D-10) | Heading: **No constitution for this project** · Body: This project doesn't have a constitution yet. Define one via the CLI, then switch back to view it here. |
| Empty state — generic view (no specs/decisions) | Heading: **Nothing here yet** · Body: This project has no {specs \| decisions} to show. |
| Error state (any project-scoped view) | **Couldn't load {view name}.** Check your connection and try again. — paired with a `Button variant="outline"` labeled **Retry**. |
| Loading | Skeleton placeholders (see State Matrix). Text fallback where skeleton is impractical: "Loading…" in `text-muted-foreground`. |
| Destructive confirmation (Revoke key — /keys migrates to shadcn even though its data-refactor is out of scope) | `AlertDialog` — Title: **Revoke this key?** · Body: This permanently disables the key. Apps using it will stop working. · Confirm button: **Revoke** (`variant="destructive"`) · Cancel: **Cancel** |

**Nav / selector states (D-08):**
- Zero projects → **no selector rendered**; main area shows the zero-projects empty state.
- Exactly one project → static label (`text-sm text-muted-foreground`), no dropdown.
- More than one → `Select` dropdown, options sorted case-insensitive alphabetical (D-05), current value bound to `project.current`.

**Active-project indicator (D-11):** every project-scoped page renders the active project name in a breadcrumb/heading zone, e.g. `Breadcrumb: {project} / {View}` where `{project}` uses `text-foreground font-semibold` and the trailing segment uses `text-muted-foreground`. Must update on project switch.

## Per-View State Matrix (D-09, D-10)

Each project-scoped view MUST render all three states. Loading uses `Skeleton`; empty and error use the copy above.

| View | Loading | Empty | Error |
|------|---------|-------|-------|
| Dashboard `/` | Skeleton stat cards (StatsBar shape) + skeleton table rows | "Nothing here yet" if zero specs & decisions | Retry card |
| Graph `/graph` | Skeleton `Card` block (graph canvas footprint) | "Nothing here yet — no graph nodes for this project." | Retry card |
| Constitution `/constitution` | Skeleton meta + section rows | **No constitution for this project** (D-10 copy) | Retry card |
| Spec detail `/spec/[slug]` | Skeleton title + metadata + section rows | "Spec not found in this project." | Retry card |
| Decision detail `/decision/[slug]` | Skeleton title + metadata | "Decision not found in this project." | Retry card |

**Project-switch behavior (visual contract for D-01):** on switch, the active-project indicator updates immediately; each view returns to its Loading (skeleton) state while `invalidateAll()` re-runs `load()`, then resolves to data / empty / error. No stale previous-project data may remain visible after the skeleton clears. Constitution's Merged/Layer + layer badges must re-derive from the new project's `provenance` (D-10).

## Interaction Contracts

| Interaction | Contract |
|-------------|----------|
| Project switch (multi) | `Select` change → `project.current` setter (writes localStorage) → `invalidateAll()`. All in-scope views re-fetch with new `X-Specgraph-Project` header. Optional `sonner` toast: "Switched to {project}". |
| Default resolution (D-04) | (1) valid last-used from localStorage → (2) project named `default` → (3) alphabetically-first. Selector reflects the resolved value on load. |
| Single project (D-08) | Static label, no interactive control. |
| Zero projects (D-07) | No selector; main area shows zero-projects empty state; nav still renders (theme toggle stays usable). |
| Theme toggle (D-14) | `mode-watcher` `toggleMode()`; persists to localStorage; no flash on reload; respects system preference on first visit. |
| Focus / a11y | All shadcn primitives keep bits-ui keyboard + focus-ring behavior; focus ring uses `--ring` (accent). Selector, toggle, and dialogs are keyboard-operable. Icon-only buttons carry `aria-label`. |
| Active-project indicator (D-11) | Breadcrumb-style `{project} / {View}` on every in-scope page; updates on switch. |
