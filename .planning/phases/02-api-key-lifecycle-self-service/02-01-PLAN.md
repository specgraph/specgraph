## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| proto contract → generated code | Message shape decisions here (presence/absence of user_id) become the security contract every downstream handler enforces |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | Self RPC request messages | high | mitigate | CreateMyAPIKey/Rotate/List/Revoke request messages define NO user_id/target field, so a caller cannot name another owner at the wire level; owner is derived from context in the handler (Plan 05) |
| T-02-02 | Tampering | gen/ committed artifacts | medium | mitigate | gen/ is produced only by `task proto` (never hand-edited; PreToolUse hook enforces); second-run byte-stability verified |
| T-02-03 | Spoofing | ResyncUserRole exposure | high | mitigate | ResyncUserRole is a distinct RPC that Plan 03 maps to `user.manage` (admin-only); no self/owner field lets a non-admin target arbitrary users |
</threat_model>
