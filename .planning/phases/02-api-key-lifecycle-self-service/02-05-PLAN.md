<objective>
Implement the four AUTH-03 self-service handlers on `IdentityHandler` (CreateMyAPIKey, ListMyAPIKeys, RotateMyAPIKey, RevokeMyAPIKey), wiring together every gate the canonical g7st design mandates: owner-from-context, `Source=="apikey"` anti-key-chaining rejection, the `RoleMin` floor on create AND rotate (SC#2), owner-scoped storage calls, the per-identity rate limiter, expiry clamp to the 90d/180d config caps (D-08), quota-safe mint, and emit-once delivery. This is the core AUTH-03 deliverable satisfying SC#1 and SC#2.

Purpose: A non-admin OIDC user can safely self-provision a role-capped, expiring key that can never launder a higher role.
Output: Four self handlers + wiring (config, rate limiter, clampExpiry) + adversarial unit tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → self RPC | Owner, role floor, and source are all enforced server-side; nothing about the target is client-trusted |
| caller identity → minted key ceiling | The role floor bounds the minted key at the caller's live effective role |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-14 | Elevation of Privilege | role_downgrade laundering (downgraded apikey mints owner-role key) | critical | mitigate | RoleMin(requestedOrInherit, caller.EffectiveRole) floor on create AND rotate; inherit resolves to EffectiveRole before RoleMin |
| T-02-15 | Elevation / Persistence | Key-chaining (leaked non-admin key self-perpetuates) | high | mitigate | Reject Source=="apikey" on CreateMyAPIKey/RotateMyAPIKey with CodePermissionDenied |
| T-02-16 | Information Disclosure | ListMyAPIKeys leaks all users' keys via empty filter | high | mitigate | Hard-set ListAPIKeysFilter.UserID from context; TestListMyAPIKeys_ScopedToCaller invariant |
| T-02-17 | Denial of Service | argon2id compute DoS via mint/rotate spam | high | mitigate | Per-identity golang.org/x/time/rate limiter → CodeResourceExhausted |
| T-02-18 | Elevation (latency) | Over-long expiry extends stale privilege | high | mitigate | clampExpiry enforces 90d default / 180d max (D-08) → CodeInvalidArgument over cap |
| T-02-19 | Information Disclosure | Secret leakage / re-serialization | medium | mitigate | Emit-once plaintext via FormatAPIKeyToken; never re-returned; label treated as untrusted in logs |
