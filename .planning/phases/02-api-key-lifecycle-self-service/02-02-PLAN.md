<objective>
Add the owner-scoped storage layer AUTH-03 needs: interface + Postgres implementations of `GetAPIKeyForUser`, `RevokeAPIKeyForUser`, `RotateAPIKeyForUser` (explicit floored/capped args), and a quota-safe `CreateAPIKeyForUser` that opens an explicit `AuthStore` transaction, locks the parent `users` row `FOR UPDATE`, counts active keys, then inserts. Add the `ErrQuotaExceeded` sentinel. Update BOTH compile-gate stub tests so the interface change stays build-green.

Purpose: Enforce ownership in the SQL WHERE clause (enumeration-hardening) and close the quota TOCTOU race at the storage boundary — the two invariants the handler cannot enforce alone.
Output: New owner-scoped `*AuthStore` methods, a new sentinel, integration tests, updated stubs.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| handler → storage | Ownership and quota are enforced in SQL, not trusted from the caller |
| concurrent mints → single users row | Serialization boundary for TOCTOU |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-04 | Information Disclosure | Cross-user key enumeration | high | mitigate | `AND user_id = $caller` WHERE + uniform `ErrAPIKeyNotFound` (RowsAffected()==0) so "not yours"/"missing"/"revoked" are indistinguishable |
| T-02-05 | Tampering / DoS | Quota TOCTOU (concurrent mints exceed quota) | high | mitigate | Explicit AuthStore tx + `SELECT 1 FROM users … FOR UPDATE` serializes a user's mints; count+insert inside the tx |
| T-02-06 | Elevation of Privilege | Rotate re-pins stale higher ceiling | high | mitigate | RotateAPIKeyForUser takes explicit floored roleDowngrade + capped expiresAt; never inherits old values |
| T-02-07 | Tampering | Invalid `count(*) FOR UPDATE` gives false safety | medium | mitigate | Parent-row lock pattern used instead; integration concurrency test asserts the cap holds |
</threat_model>
