## 3. Cross-Phase Integration (7/7 seams verified)

Verified by `gsd-integration-checker` against the live codebase (static wiring review, evidence-backed):

1. **Identity/auth convergence (P2↔P3):** `resolveJWT`, `ResolveLogin`, and `resolveIntrospection` all funnel through a single `materializeIdentity` binding→JIT→soft-delete→login-sync tail (`internal/auth/identitystore.go:513,522,229`). No divergent identity paths.
2. **Shared role/floor machinery (P2↔P3):** self-mint create/rotate uses `auth.RoleMin` (`identity_handler.go:576`, `rolemin.go:14`); standing keys use `clampedRole` (`identitystore.go:398,341`); `ResyncUserRole` writes `users.role` via the same `UpdateUserRole` seam so live floor applies to standing keys.
3. **Single canonical issuer:** one `ProviderIssuer` helper (`config/global.go:333`, 9 callers) feeds oauth2 `claims.Issuer`, claims-mapping key, RFC 9728 `authorization_servers`, and session issuer — cannot diverge (AUTH-05/D-09).
4. **Static-credential guard before introspection:** `Resolve` routes `spgr_ws_`/`spgr_sk_` prefixes ahead of the introspection branch (`identitystore.go:208-223`) — no API-key secret is POSTed to an external IdP.
5. **Handler registration:** `serve.go:246-268` registers Identity/Lifecycle/etc.; `ResyncUserRole`/`CreateMyAPIKey`/`CheckDrift`/`AcknowledgeDrift` are real impls on registered handlers (no defined-but-unregistered handler).
6. **CSRF double-submit full loop (AUTH-03):** server issues `specgraph_csrf` on whoami GET (`csrf.go:108`), validates on the 4 self-key POSTs exempting Bearer (`csrf.go:28-60`); web `csrfInterceptor` echoes the cookie into `X-CSRF-Token` (`web/src/lib/api/client.ts:44-48`). Names match byte-for-byte.
7. **Web↔API project scoping + MCP drift + config/release:** `projectInterceptor` sends `X-Specgraph-Project` on every RPC; project switch calls `invalidateAll()` re-running `+page.ts` loads (skeleton-on-switch); MCP `drift` tool → `LifecycleService.CheckDrift/AcknowledgeDrift`; `runServe` consumes the koanf loader; single `goreleaser release --clean` job (no dual publish path).
