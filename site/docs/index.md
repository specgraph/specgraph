# SpecGraph

**Live Spec-Driven Development Framework**

Manage software specifications as a queryable graph — not static markdown files.
SpecGraph provides a constitution, spec schema, authoring process, and live
storage + query layer that feeds execution engines.

---

## The Problem

Existing SDD frameworks treat specs as static documents with no queryability,
addressability, or execution interface. SpecGraph closes those gaps.

---

## Core Concepts

<div class="grid cards" markdown>

- :material-graph: **Specs as Graph Nodes**

  ---

  Dependencies, blocks, and compositions are first-class edges — not filename references.

- :material-layers: **Progressive Structure**

  ---

  Solo dev uses 3 fields. Enterprise adds governance layers. Same schema.

- :material-shield-check: **Constitution**

  ---

  Layered project ground truth: languages, principles, constraints, patterns.

- :material-robot: **Agent-Native**

  ---

  Every spec is a machine-readable work unit that agents can claim, execute, and verify.

</div>
