# spgr-8ar Piece E â€” Prime Unification

> **For agentic workers:** Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Collapse the three drifted "prime" surfaces (`ExecutionService.GetPrime` RPC, `specgraph://prime` MCP resource, `specgraph prime` CLI) onto a single shared composer in a new `internal/prime` package, with two scopes (project, spec). Add `specgraph://prime/spec/{slug}` as a new templated MCP resource. Extend the CLI with an optional `{slug}` positional and `--show-provenance` + `--json` flags. Preserve the existing `runUp` side effect (Claude Code SessionStart hook depends on it).

**Architecture:** New `internal/prime` package owns composition; each of the three surfaces becomes a thin adapter (handler / resource handler / cobra command). Renderers live separately so markdown rendering is shared between MCP resource and CLI. Proto gains a `view` oneof on `PrimeResponse` carrying typed `ProjectView` / `SpecView` messages (field numbers 10, 11). Existing summary fields (1â€“5) preserved for backward compat, populated only for spec scope. `GetPrimeRequest.slug` becomes optional (empty â†’ project view).

Design reference: `docs/superpowers/specs/2026-05-21-multi-layer-constitution-completion-design.md` Section 10 (Prime Unification â€” Piece E) and Section 14 invariants `view oneof` and `CLI byte-stability`.
